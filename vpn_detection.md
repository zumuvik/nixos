## 1. Назначение
Методика описывает алгоритмы выявления VPN и Proxy на устройствах пользователей для минимизации обхода ограничений РКН. Включает в себя анализ сетевых сессий, интерфейсов и системных таблиц маршрутизации.

## 2. Термины и сокращения
| Термин | Определение |
| :--- | :--- |
| **VPN** | Зашифрованный туннель для перенаправления трафика. |
| **Proxy** | Посредник на уровне приложений (HTTP, SOCKS). |
| **GeoIP** | Определение локации по IP через базы данных или API. |
| **ASN** | Autonomous System Number — владелец сети/провайдер. |
| **MTU** | Maximum Transmission Unit — максимальный размер пакета. |

---

## 3. Этапность внедрения

| Этап | Название | Суть метода |
| :--- | :--- | :--- |
| **1** | **GeoIP** | Проверка IP на сервере по репутационным базам. |
| **2а** | **Direct Mobile** | Прямые системные признаки в Android/iOS. |
| **2б** | **Accuracy** | Косвенные признаки (MTU, имена интерфейсов). |
| **3** | **Desktop** | Анализ на Windows, macOS и Linux. |

---

## 4. Ложноположительные срабатывания
**Внимание:** признаки VPN могут быть выявлены в легитимных случаях:
*   **Корпоративные сети:** Рабочий VPN для доступа в интранет.
*   **Виртуализация:** Docker, WSL2, VirtualBox создают свои интерфейсы.
*   **Безопасность:** Антивирусы и фаерволы часто используют локальные туннели.
*   **NAT:** Особенности маршрутизации провайдеров.

## 5. Технические методы

### 🤖 Android
Для детекции используются API `ConnectivityManager` и `NetworkCapabilities`.
**Прямые признаки:**
- `IS_VPN` в политиках.
- Транспорт `TRANSPORT_VPN`.
- Наличие пакета в `VpnTransportInfo`.
