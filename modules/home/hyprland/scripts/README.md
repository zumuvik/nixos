# Документация скриптов Hyprland

## Общие паттерны

Все скрипты следуют одинаковым паттернам и соглашениям:

### 1. Shebang и строгий режим
```bash
#!/usr/bin/env bash
set -euo pipefail
```

### 2. Переменные конфигурации
```bash
iDIR="${HOME}/.config/swaync/icons"
SCRIPTSDIR="${HOME}/.config/hypr/scripts"
```

### 3. Паттерн case для обработки аргументов
Все скрипты используют `case` для разбора аргументов командной строки:
```bash
case "$1" in
    "--get")  get_value ;;
    "--inc")  increment ;;
    "--dec")  decrement ;;
    *)        get_value ;;
esac
```

### 4. Уведомления
Большинство скриптов используют `notify-send` с синхронизацией для единообразного интерфейса:
```bash
notify-send -e -h string:x-canonical-private-synchronous:volume_notif \
            -h int:value:"$level" \
            -u low -i "$icon" \
            "Сообщение" "Содержание"
```

## Категории скриптов

### 1. Дисплей и яркость

#### Brightness.sh
Управление яркостью дисплея через `brightnessctl`.

**Действия:**
- `--get`: Получить текущий процент яркости
- `--inc`: Увеличить яркость на шаг (по умолчанию: 10%)
- `--dec`: Уменьшить яркость на шаг (по умолчанию: 10%)

**Особенности:**
- Автоматическое масштабирование на интервалы 20% для выбора подходящих иконок
- Уведомление с иконкой, указывающей уровень яркости
- Ограничение яркости от 5% до 100%

**Пример использования:**
```bash
Brightness.sh --inc    # Увеличить на 10%
Brightness.sh --dec    # Уменьшить на 10%
Brightness.sh          # Показать текущее значение
```

#### Hyprsunset.sh
Управление цветовой температурой экрана (ночной свет).

**Действия:**
- `toggle`: Включить/выключить ночной свет
- `status`: Вывести JSON для статуса Waybar
- `init`: Инициализировать с сохранённым состоянием

**Конфигурация:**
Установить переменные окружения:
- `HYPRSUNSET_TEMP`: Целевая температура в Кельвинах (по умолчанию: 4500)
- `HYPRSUNSET_ICON_MODE`: Режим отображения иконки (`sunset` или `blue`)

**Особенности:**
- Сохраняет состояние в `$HOME/.cache/.hyprsunset_state`
- Предотвращает конфликты с другими менеджерами цветовой температуры
- Поддерживает разметку Pango для цветных иконок в Waybar

---

### 2. Управление звуком

#### Volume.sh
Управление громкостью системы и микрофона.

**Действия:**
- `--get`: Получить статус громкости (Muted или процент)
- `--inc`: Увеличить громкость на 5%
- `--dec`: Уменьшить громкость на 5%
- `--inc-precise`: Увеличить на 1%
- `--dec-precise`: Уменьшить на 1%
- `--toggle`: Переключить режим без звука
- `--toggle-mic`: Переключить режим без звука микрофона
- `--get-icon`: Получить иконку громкости
- `--get-mic-icon`: Получить иконку микрофона
- `--mic-inc`: Увеличить громкость микрофона
- `--mic-dec`: Уменьшить громкость микрофона

**Особенности:**
- Контекстные иконки (без звука, низкая, средняя, высокая)
- Раздельное управление динамиками и микрофоном
- Звуковая обратная связь при изменении громкости
- Интеграция со SwayNC для обхода режима "не беспокоить"

### 3. Снимки экрана и медиа

#### ScreenShot.sh
Комплексный инструмент для снятия скриншотов с использованием `grim` и `wl-copy`.

**Действия:**
- `--now`: Сделать снимок немедленно
- `--in5`: Сделать снимок через 5-секундный обратный отсчёт
- `--in10`: Сделать снимок через 10-секундный обратный отсчёт
- `--win`: Снимок активного окна
- `--area`: Выбрать область для снимка
- `--active`: Снимок с классом окна в названии файла
- `--swappy`: Редактировать снимок в Swappy немедленно

**Особенности:**
- Автоматическое именование файлов с временными метками
- Интеграция с буфером обмена (wl-copy)
- Уведомление с кнопками действий (Открыть/Удалить)
- Звуковые эффекты
- Таймер обратного отсчёта для отложенных снимков
- Поддержка как сохранения, так и редактирования рабочих процессов

#### MediaCtrl.sh
Управление медиаплеером через `playerctl`.

**Действия:**
- `--nxt`: Следующий трек
- `--prv`: Предыдущий трек
- `--pause`: Переключить воспроизведение/паузу
- `--stop`: Остановить воспроизведение

**Особенности:**
- Отображает метаданные трека в уведомлениях
- Поддерживает всех игроков, совместимых с playerctl
- Системное управление медиа

### 4. Управление системой

#### LockScreen.sh
Блокирует сеанс с помощью `loginctl`.

**Особенности:**
- Гарантирует обновление кэша погоды перед блокировкой
- Работает с Hyprlock (в настоящее время закомментирован)
- Чистая блокировка сеанса

#### Wlogout.sh
Запускает меню выключения wlogout с адаптивным размером.

**Особенности:**
- Автоматически определяет разрешение и масштаб монитора
- Подстраивает размер кнопок под разрешение экрана
- Работает как протокол layer-shell на всех разрешениях
- Поддерживает дисплеи от 720p до 4K

**Размеры на основе разрешения:**
- ≥2160p: кнопки 600px
- 1600–2160p: кнопки 400px
- 1440–1600p: кнопки 400px
- 1080–1440p: кнопки 200px
- 720–1080p: кнопки 50px

#### Скрипты питания и сна
- `Hypridle.sh`: Переключить демон hypridle
- `GameMode.sh`: Отключить анимацию для игр
- `Animations.sh`: Настроить предустановки анимации

### 5. Управление окнами

#### Dropterminal.sh
Выпадающий терминал с плавной анимацией.

**Особенности:**
- Скользящая анимация показа/скрытия
- Запоминание позиции для каждого монитора (сохраняет имя монитора)
- Независимое от рабочего пространства функционирование
- Настраиваемые размер и позиция (в процентах)

**Конфигурация:**
```bash
WIDTH_PERCENT=65   # Ширина окна %
HEIGHT_PERCENT=65  # Высота окна %
Y_PERCENT=10       # Вертикальная позиция от вершины
```

**Действия:**
- Запускает терминал в специальном рабочем пространстве (scratchpad)
- Анимирует появление с верхней части экрана
- Анимирует скрытие за верхнюю часть экрана
- Переключает между видимым и скрытым состояниями
- Правильно обрабатывает смену мониторов

**Использование:**
```bash
Dropterminal.sh kitty                  # Запустить с kitty
Dropterminal.sh -d foot                # С выводом отладки
Dropterminal.sh "kitty -e zsh"         # Пользовательская команда
```

#### KeyHints.sh
Отображает все сочетания клавиш через `yad`.

Показывает список всех настроенных привязок клавиш из файлов конфигурации.

#### OverviewToggle.sh
Вызывает AGS (если установлен) или rofi для обзора рабочего стола.

### 6. Тема и внешний вид

#### DarkLight.sh
Полнофункциональный переключатель темы между светлым и тёмным режимами.

**Переключаемые элементы:**
- Стили Waybar (`[Dark].css` / `[Light].css`)
- Фон уведомлений SwayNC
- Стили AGS (если установлен)
- Цвета терминала Kitty
- Цвета Rofi
- Цветовые схемы Qt5CT/Qt6CT
- Тема Kvantum
- Тема и иконки GTK
- Палитра темы Wallust
- Динамические обои (папки Dark/Light)
- Тема эмодзи Rofi

**Рабочий процесс:**
1. Определяет текущий режим из `$HOME/.cache/.theme_mode`
2. Переключается на противоположный режим
3. Обновляет все файлы конфигурации
4. Перезагружает затронутые процессы (waybar, rofi, swaync, ags, swaybg)
5. Обновляет обои
6. Уведомляет пользователя об изменении

**Требования:**
- `wallust` для генерации цветовой схемы
- Обои в `$PICTURES_DIR/wallpapers/Dynamic-Wallpapers/Dark` и `Light`
- Темы GTK в `$HOME/.themes` с правильным именованием
- Иконки в `$HOME/.icons`

**Использование:**
```bash
DarkLight.sh    # Переключить между режимами
```

#### ThemeChanger.sh
Простой переключатель обоев с использованием `awww`.

#### WallustSwww.sh
Применить цвета wallust к awww как эффекты обоев.

#### WaybarScripts.sh & WaybarLayout.sh & WaybarStyles.sh
Настроить внешний вид и позицию Waybar.

#### Refresh.sh & RefreshNoWaybar.sh
Перезагрузить затронутые приложения:
- waybar, rofi, swaync, ags
- Опция пропуска обновления waybar

## Инструкции по использованию

### Добавление нового скрипта

1. **Создайте скрипт** в каталоге `scripts/` с расширением `.sh`
2. **Сделайте исполняемым:**
    ```bash
    chmod +x scripts/YourScript.sh
    ```
3. **Добавьте в Home Manager:**
    ```nix
    home.file.".config/hypr/scripts/YourScript.sh".source = ./scripts/YourScript.sh;
    ```
4. **Настройте привязку клавиш** в `binds.nix` или `Keybinds.conf`
5. **Протестируйте отдельно:**
    ```bash
    ./.config/hypr/scripts/YourScript.sh --option
    ```

### Тестирование скриптов

Перед интеграцией с привязками клавиш:
```bash
# Тест с конкретными аргументами
./scripts/Volume.sh --inc

# Проверить вывод
./scripts/Brightness.sh --get

# Режим отладки
./scripts/Dropterminal.sh -d foot
```

### Интеграция с привязками клавиш

В `modules/home/hyprland/binds.nix`:
```nix
{
  bind = [
    "SUPER, b, exec, $HOME/.config/hypr/scripts/Brightness.sh --inc"
    "SUPER+SHIFT, b, exec, $HOME/.config/hypr/scripts/Brightness.sh --dec"
    "SUPER, space, exec, $HOME/.config/hypr/scripts/Dropterminal.sh kitty"
  ];
}
```

Или в `Keybinds.conf`:
```conf
$mod = Super
bind = $mod, b, exec, $HOME/.config/hypr/scripts/Brightness.sh --inc
```

## Интеграция с Hyprland

### Структура файлов

```
/etc/nixos/
├── flake.nix
├── modules/
│   └── home/
│       └── hyprland/
│           ├── hyprland.nix     # Импортирует scripts.nix
│           ├── scripts.nix      # Копирует скрипты в конфиг
│           └── binds.nix        # Определения привязок клавиш
```

### Путь импорта

Скрипты копируются через Home Manager:
```nix
# scripts.nix
{
  home.file.".config/hypr/scripts".source = ./scripts;
  home.file.".config/hypr/UserScripts".source = ./UserScripts;
}
```

Это создаёт: `$HOME/.config/hypr/scripts/`

### Файлы конфигурации

Скрипты ссылаются на конфигурацию в:
- `$HOME/.config/hypr/UserConfigs/` - Переопределения пользователя
- `$HOME/.config/hypr/UserAnimations/` - Предустановки анимации
- `$HOME/.cache/` - Состояние во время выполнения

## Решение проблем

### Типичные проблемы

#### Ошибка "Script not found"
- Убедитесь, что скрипт находится в каталоге `scripts/`
- Проверьте, что Home Manager импортирует `scripts.nix`
- Проверьте путь: `$HOME/.config/hypr/scripts/`

#### Уведомление не отображается
- Проверьте, что `notify-send` установлен
- Убедитесь, что SwayNC или демон уведомлений запущен
- Проверьте пути иконок: `$HOME/.config/swaync/icons/`

#### Ошибка "Permission Denied"
```bash
chmod +x scripts/YourScript.sh
```

#### Скрипт не запускается
- Добавьте `set -euo pipefail` для строгого режима
- Проверьте наличие зависимостей:
    ```bash
    which jq rofi grim slurp
    ```
- Используйте режим отладки, если доступен:
    ```bash
    Dropterminal.sh -d foot
    ```

#### Интеграция Waybar не работает
- Убедитесь, что Waybar перезагружается после изменения конфигурации:
    ```bash
    pkill waybar && waybar &
    ```
- Проверьте формат JSON для пользовательских модулей
- Поищите ошибки синтаксиса в конфигурации Waybar

#### Проблемы с интеграцией SwayNC
- Установите `swaync`: `nixpkgs#swaync`
- Включите в Home Manager:
    ```nix
    services.swaync.enable = true;
    ```
- Проверьте, что сеанс D-Bus запущен

### Режим отладки

Для сложных скриптов:
```bash
# Включить отладку bash
bash -x scripts/ScriptName.sh --arg

# Использовать strace для системных вызовов
strace -f scripts/ScriptName.sh --arg

# Проверить окружение
env | grep -E '(HOME|XDG|HYPR)'
```

### Логи и файлы состояния

Скрипты хранят состояние в:
- `$HOME/.cache/.theme_mode` - Текущая тема
- `$HOME/.cache/.hyprsunset_state` - Статус ночного света
- `/tmp/dropdown_terminal_addr` - Адрес окна терминала
- `/tmp/hypr_keybind_suggestions_file` - Предложения привязок

### Отладка зависимостей скрипта

```bash
# Проверить все требуемые инструменты
for cmd in jq rofi grim slurp wl-copy; do
    if command -v $cmd >/dev/null 2>&1; then
        echo "✓ $cmd доступен"
    else
        echo "✗ $cmd отсутствует"
    fi
done
```

## Лучшие практики

### 1. Обработка ошибок
Всегда используйте строгий режим:
```bash
set -euo pipefail
```

### 2. Пути файлов
Используйте `$HOME` вместо абсолютных путей:
```bash
# Хорошо
config_file="$HOME/.config/hypr/config.conf"

# Плохо
config_file="/home/user/.config/hypr/config.conf"
```

### 3. Уведомления
Используйте единообразное форматирование:
```bash
notify-send -e -u low -i "$iDIR/icon.png" \
            -h string:x-canonical-private-synchronous:notif_id \
            "Категория" "Сообщение"
```

### 4. Исполняемость
Все скрипты должны быть исполняемыми:
```bash
chmod +x scripts/*.sh
```

### 5. Идемпотентность
Скрипты должны правильно обрабатывать многократные вызовы:
```bash
if pgrep -x "process" >/dev/null; then
    pkill "process"
fi
```

### 6. Логирование
Логируйте в файлы для отладки:
```bash
exec >> "$HOME/.config/hypr/scripts/debug.log" 2>&1
set -x  # Трассировка выполнения
```
